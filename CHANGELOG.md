### Added
- **Android Power Tools module** (new, sort order 2, sits between Android Devices and Jellyfin in the sidebar). Hosts ws-scrcpy-web's full home page via iframe at `/android-power-tools` â€” one-click shell (xterm), file browser (ListFilesModal), stream configuration, network scan + manually-add-device panel, and dependency updater. Peer of the Android Devices module, not a replacement â€” the existing device management surface (registered-devices list, PIN unlock, power state, sleep/wake, screensaver, Projectivy backups) stays untouched. Module declares no dependencies of its own (inherits ws-scrcpy-web from AndroidDevices). Fallback: if `WsScrcpy.IsRunning` is false, shows a warning alert instead of a broken iframe.
- `CHANGELOG.md` (this file) following Keep a Changelog format.
- `LICENSE` â€” GPL-3.0-only (matches ws-scrcpy-web, the primary external dependency).
- `SECURITY.md` â€” private vulnerability reporting flow via GitHub Security Advisories, 72-hour acknowledgement SLA.
- `CONTRIBUTING.md` â€” setup, build/test commands, project structure, module architecture, code style, PR guidelines.
- `.gitignore` entries for `.claude/` (Claude Code session artifacts), `src/ControlMenu/dependencies/go2rtc/*` (runtime-managed binaries, matching existing pattern for other dep folders), and `src/ControlMenu/go2rtc.yaml` (runtime-generated config rebuilt on startup from the cameras table).
- `DeviceType` enum gained `AndroidTablet` and `AndroidWatch` alongside the existing `GoogleTV` / `AndroidPhone`. Persisted as string via EF so existing rows are untouched; no migration needed. Tablet and watch devices don't yet get a dedicated dashboard, but they're first-class in the Device Management table and the mDNS scan auto-classifier.
- `ScrcpyMirror.razor` gained an optional `DeviceKind` parameter (`"phone"`, `"tablet"`, or `"tv"`). When set, it's forwarded to ws-scrcpy-web's `/embed.html` as `&deviceKind=<value>` so the stream toolbar's D-pad/Touch input-mode toggle seeds the right default. `GoogleTvDashboard.razor` passes `"tv"` (D-pad default); `PixelDashboard.razor` passes `"phone"` (Touch default). Requires ws-scrcpy-web commit `6f217cd` or later, which wires the URL param through in `embed-entry.ts`.
- `IAdbService.ScanMdnsAsync()` returns a list of `MdnsAdbDevice` records (service name + IP + advertised ADB port) by parsing `adb mdns services`. Uses a direct port of ws-scrcpy-web's tab-split `parseMdnsOutput` for robustness.
- `IAdbService.DetectDeviceKindAsync(ip, port)` classifies an Android device as `"phone"`, `"tablet"`, `"tv"`, or `"watch"` using five parallel shell probes (`ro.build.characteristics`, `pm has-feature android.software.leanback`, `pm has-feature android.hardware.type.watch`, `wm size`, `wm density`). Mirrors ws-scrcpy-web's four-signal `classifyDeviceKind` plus a fifth watch-feature probe for Wear OS support. Watch precedence > TV > tablet/phone smallestWidthDp split.
- `IAdbService.GetPropAsync(ip, port, property)` â€” thin `adb shell getprop <name>` wrapper returning the trimmed value (or empty string on failure). Used by the Discovered panel to pre-fill a newly-added device's Name from `ro.product.model` when no name is remembered.
- **"Discovered on Network" panel** in Settings > Devices. After a Scan, mDNS-advertised devices that aren't yet registered show up in a dedicated table with IP, advertised ADB port, MAC (resolved via ARP â€” a ping pass refreshes ARP for any mDNS IPs not already in the table), and a one-click Add button. Clicking Add opens the Device form pre-filled with MAC, IP, port, and an initial Type of `AndroidPhone`; a background ADB probe then refines Type (`DetectDeviceKindAsync`) and auto-fills Name from `ro.product.model` when the name wasn't already restored from the delete-remembers-name flow.
- **Delete remembers the device's name.** When a device is deleted from Settings > Devices, its user-assigned name is stashed in the Settings table at `device-name-<mac>`. When the same MAC later reappears via mDNS scan and the user clicks Add, the form pre-fills Name from that stashed value before falling back to the ADB product-model probe â€” the user doesn't retype "My Phone" just because they deleted and re-added the device.
- **Network scanner modal** â€” `Settings â€ş Devices` gained a new *Scan Networkâ€¦* button that opens a full scanner modal (CIDR/IP/range subnet input, gateway auto-detect, four-state progress chip, live hit stream, Add buttons per hit, 2,048-host large-subnet warning, subnet syntax cheat sheet at `/help/subnets.html`).
- **ws-scrcpy-web deployment mode** â€” General Settings gained a Managed / External toggle. Managed keeps today's auto-spawn behaviour; External lets Control Menu point at a Docker container or remote host via URL. Dependency management respects mode: Install/Update hidden in External mode.
- **`NetworkScanService` (singleton)** â€” holds a `ClientWebSocket` to ws-scrcpy-web's `/ws-scan` endpoint, fans scan events to every Blazor circuit spectating a scan. Two browser tabs attaching mid-scan share one scan session with snapshot replay.
- **`SubnetParser`** â€” CIDR / IP / range parser mirroring ws-scrcpy-web's behaviour byte-for-byte, error verbiage included.

### Changed
- `MainLayout.razor` page-title switch gained a specific case for `/android-power-tools` that sits above the generic `path.StartsWith("android")` fallback. Without this the breadcrumb rendered "Android Devices" for the new route because the prefix match order is load-bearing.
- **Scan Network** (Settings > Devices) now queries mDNS first via `adb mdns services`, enriches each hit with MAC from the ARP table (pinging any mDNS IPs not yet in ARP to force resolution), and falls back to ARP-only refresh for devices that don't advertise over mDNS. When a device is matched via mDNS, Control Menu updates both its IP address AND its stored ADB port â€” previously the port would remain stale when a device's wireless debugging port changed (common on Android 11+ phones that rotate ports per session). Discovered unregistered devices are surfaced in a new panel below the registered-devices table (see Added). Toast reports `N of M registered device(s) found`, `K ADB port(s) updated from mDNS`, and `X new device(s) on network` as applicable.
- **Adding a new device auto-removes it from the Discovered panel.** Without this, the user would see the device they just added still listed as "available to add" until the next scan.
- `ScrcpyMirror.razor` now uses ws-scrcpy-web's `/embed.html?device=<udid>` wrapper URL (shipped 2026-04-17 in ws-scrcpy-web 1.0.0); the legacy `#!action=stream&udid=...&embed=true` hash routing has been removed upstream. ### Removed
- `UsbSetupWizard.razor` and its stylesheet. The component existed to wire a freshly-flashed Android phone to wireless ADB over a USB cable (`adb tcpip <port>` â†’ disconnect USB â†’ connect over WiFi). ws-scrcpy-web's mDNS-based discovery replaces this flow end-to-end â€” phones advertise their ADB port over mDNS once wireless debugging is enabled once by the user, so the USB handshake dance is no longer necessary. The corresponding "Enable Wireless ADB" action-row on `PixelDashboard.razor` and the new-phone auto-trigger on `DeviceManagement.razor`'s Save flow have been removed. `IAdbService.GetUsbDevicesAsync` and `IAdbService.ResetTcpPortAsync` were deleted along with their implementations â€” both existed only to support the wizard.
